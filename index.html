üìö Introduction

The DonnySavage Bot API provides secure, programmatic access to bot functionality and server data. This RESTful API allows developers to integrate bot features into their applications and services.

Tip: All API endpoints require authentication. Make sure to include your API key in the x-api-key header.

API Base URL

http://bot.api.donnysavagebot.publicvm.com

Authentication

All endpoints require API key authentication:

x-api-key: YOUR_API_KEY

Keys are generated with /auth generate-key

üîë Getting an API Key

API keys are generated through the bot's commands:

/auth generate-key service="My Web Service"

Each key is tied to a specific Discord server and grants access to that server's resources.

Security Notice: API keys provide access to sensitive server data. Never expose them in client-side code!

üîå API Endpoints

GET
/api/secure-data

Verifies API key authentication and returns basic service information.

Use Case: First endpoint to call to verify your API key is working properly
Rate Limit: 30 requests per 10 minutes per IP
Response
{
  "status": "success",
  "message": "Access granted to protected resource",
  "service": "Your Service Name",
  "guildId": "123456789012345678"
}

Response Fields

Field Type Description
status string Operation status ("success")
message string Confirmation message
service string Name of the service associated with the API key
guildId string Discord server ID associated with the API key
GET
/api/bot-stats

Returns real-time statistics about the bot's presence and performance.

Use Case: Monitor bot health and performance on your dashboard
Rate Limit: 30 requests per 10 minutes per IP
Response
{
  "guilds": 127,
  "users": 38426,
  "uptime": 12345.67
}

Response Fields

Field Type Description
guilds integer Number of servers where bot is active
users integer Total unique users across servers
uptime float Bot uptime in seconds
POST
GET
/api/verify-member

Checks if a user belongs to your server, returns member information, or gets server member count.

Use Case: Authenticate users for role-based access or restrict services to Discord members. For instance, allow access only to users who log in with their Discord account and belong to your server, ensuring secure and exclusive content delivery.
New Use Case: Display server member count on your website or dashboard to show community size
Rate Limit: 30 requests per 10 minutes per IP

Request Parameters (GET)

Parameter Type Description
userId string Discord user ID or "count" to get server member count

Request Body (POST)

{
  "userId": "987654321098765432" // or "count" for member count
}
Response (Member Verification)
{
  "id": "987654321098765432",
  "username": "DiscordUser",
  "discriminator": "1234",
  "avatar": "https://cdn.discordapp.com/avatars/...",
  "roles": [
    {
      "id": "112233445566778899",
      "name": "Member"
    },
    {
      "id": "998877665544332211",
      "name": "Admin"
    }
  ],
  "joinedAt": "2023-01-15T10:30:00.000Z"
}

Response Fields (Verification)

Field Type Description
id string Discord user ID
username string Discord username
discriminator string User's 4-digit discriminator
avatar string URL to user's avatar image
roles array Array of role objects (id and name)
joinedAt string ISO 8601 timestamp of when user joined the server
Response (Member Count)
{
  "status": "success",
  "guildId": "123456789012345678",
  "guildName": "My Awesome Server",
  "memberCount": 250
}

Response Fields (Count)

Field Type Description
status string Operation status ("success")
guildId string Discord server ID associated with the API key
guildName string Name of the Discord server
memberCount integer Total number of members in the server

Response (Errors)

  • 400 Bad Request: Invalid/missing userId in request body
  • 401 Unauthorized: Missing or invalid API key
  • 404 Not Found: Guild or user not found
  • 429 Too Many Requests: Rate limit exceeded

// Member Verification Example
const verifyMember = async (userId, getCount = false) => {
  try {
    const bodyData = getCount ? { userId: 'count' } : { userId };
    
    const response = await fetch('http://bot.api.donnysavagebot.publicvm.com/api/verify-member', {
      method: 'POST',
      headers: {
        'x-api-key': 'YOUR_API_KEY',
        'Content-Type': 'application/json'
      },
      body: JSON.stringify(bodyData)
    });

    if (!response.ok) {
      const errorData = await response.json();
      throw new Error(`API Error: ${errorData.error || response.status}`);
    }

    const result = await response.json();
    
    // Add verification message for member lookup
    if (!getCount) {
      console.log('‚úÖ Yes Member is in the Server');
    }
    
    return result;
  } catch (error) {
    console.error('Verification failed:', error.message);
    throw error;
  }
};

// Usage examples:
// 1. Verify specific member
verifyMember('USER_ID_HERE')
  .then(memberData => {
    console.log('Member details:', memberData);
    if (memberData.roles.some(role => role.name === 'Admin')) {
      console.log('User is an admin');
    }
  })
  .catch(error => {
    console.error('Member verification failed:', error.message);
  });

// 2. Get member count
verifyMember(null, true)
  .then(data => {
    console.log(`‚úÖ The API origin server currently has ${data.memberCount} members.`);
  });

// Bot Statistics Example
const getBotStats = async () => {
  try {
    const response = await fetch('http://bot.api.donnysavagebot.publicvm.com/api/bot-stats', {
      headers: {
        'x-api-key': 'YOUR_API_KEY'
      }
    });
    
    if (!response.ok) throw new Error(`HTTP error! status: ${response.status}`);
    
    return await response.json();
  } catch (error) {
    console.error('Failed to get bot stats:', error);
  }
};

// Usage:
getBotStats().then(stats => {
  console.log(`Bot Stats: ${stats.guilds} servers, ${stats.users} users`);
});

Python

import requests
import json

# Replace with your actual API key
API_KEY = 'YOUR_API_KEY'

def verify_member(user_id=None, get_count=False):
    """
    Verify a Discord member or get server member count
    
    :param user_id: Discord user ID (string)
    :param get_count: Set True to get member count instead
    :return: API response as dictionary
    """
    try:
        # Prepare request data
        headers = {
            'x-api-key': API_KEY,
            'Content-Type': 'application/json'
        }
        
        # Handle count vs member verification
        payload = {'userId': 'count'} if get_count else {'userId': user_id}
        
        # Make API request
        response = requests.post(
            'http://bot.api.donnysavagebot.publicvm.com/api/verify-member',
            headers=headers,
            json=payload
        )
        
        # Check for errors
        response.raise_for_status()
        
        # Process successful response
        result = response.json()
        
        # Print verification success for member lookup
        if not get_count:
            print('‚úÖ Yes Member is in the Server')
        
        return result
        
    except requests.exceptions.HTTPError as http_err:
        # Handle HTTP errors (4xx, 5xx)
        error_data = response.json() if response.content else {}
        error_msg = error_data.get('error', f'HTTP Error {response.status_code}')
        print(f'‚ùå Verification failed: {error_msg}')
        raise
    except requests.exceptions.RequestException as err:
        # Handle network errors
        print(f'‚ùå Network error: {str(err)}')
        raise
    except json.JSONDecodeError:
        print('‚ùå Invalid JSON response from server')
        raise

def get_bot_stats():
    """Get bot statistics"""
    try:
        response = requests.get(
            'http://bot.api.donnysavagebot.publicvm.com/api/bot-stats',
            headers={'x-api-key': API_KEY}
        )
        response.raise_for_status()
        return response.json()
    except requests.exceptions.RequestException as err:
        print(f'‚ùå Failed to get bot stats: {str(err)}')
        raise

if __name__ == '__main__':
    # Example 1: Verify specific member

üìù Usage Guidelines

Rate Limiting

Max 30 requests per 10 minutes per IP

Note: This rate limit applies to all protected API endpoints
Tip: Implement caching for frequently accessed data to stay within limits

Error Handling

  • 401 Unauthorized: Invalid/missing API key
  • 403 Forbidden: Key doesn't have access to requested resource
  • 404 Not Found: Resource not available
  • 429 Too Many Requests: Rate limit exceeded
  • 500 Internal Server Error: Bot/server issues

Security Recommendations

  • Always use HTTPS in production environments
  • Never expose API keys in client-side code
  • Rotate API keys periodically using /auth generate-key
  • Store API keys securely using environment variables or secret managers
  • Revoke compromised keys immediately with /auth revoke-key
  • Restrict key permissions to only necessary endpoints
Warning: The verify-member endpoint can be rate limited. Implement caching for repeated requests for the same user.
